SEÑAS APP — CONTEXTO COMPLETO PARA IA
======================================

1. PROPÓSITO
   App web PÚBLICA para aprender lengua de señas mediante juegos interactivos
   (flashcards, quiz, memoria, unir). Sin login de alumnos — sesiones anónimas.
   Solo el profesor/admin necesita autenticarse para gestionar contenido.

2. ARQUITECTURA
   Frontend (Rediseño): React 18 + Vite 6 + Tailwind v4 + shadcn/ui (Rediseño de página web/) → localhost:5173
   Backend:             Express 5 + MongoDB/Mongoose (modulo-palabras/) → localhost:3001
   BD:                  MongoDB Atlas (nube)

4. MODELOS DE DATOS (MongoDB/Mongoose)

   Sesion (sesiones anónimas de niños — modelo principal para stats):
   {
     _id, sessionId (UUID único), deviceId,
     nombreAlumno (opcional), avatar (opcional),
     palabras: [{ palabraId, acertada, juegoTipo, timestamp }],
     juegosCompletados: [{ tipo, categoriaId, total, correctas, timestamp }]
   }

   Usuario (solo para profesores/admin):
   { _id, nombre, rol (superadmin|profesor), pin_hash (bcrypt), deletedAt }

   Categoria:  { _id, nombre, icono, descripcion, deletedAt }
   Nivel:      { _id, nombre, orden, categoria_id (ref), config: { max_repasos_diarios, max_tarjetas_nuevas }, deletedAt }
   Flashcard:  { _id, palabra, descripcion, imagen_url, video_url, categoria_id (ref), nivel_id (ref), creado_por (ref), deletedAt }

5. API ENDPOINTS — COMPLETO

   PÚBLICOS (sin autenticación):
     GET  /api/public/categorias           → todas las categorías
     GET  /api/public/niveles              → ?categoria_id (filtro opcional)
     GET  /api/public/palabras             → ?categoria_id&nivel_id&busqueda&page&limit (paginado)
     GET  /api/public/palabras/:id         → detalle de una palabra

   SESIONES ANÓNIMAS (sin autenticación):
     POST /api/sesiones                    → crear/actualizar sesión { sessionId, deviceId }
     POST /api/sesiones/palabra            → registrar intento { sessionId, palabraId, acertada, juegoTipo }
     POST /api/sesiones/juego              → registrar juego completado { sessionId, tipo, categoriaId, total, correctas }
     GET  /api/sesiones/:sessionId         → obtener stats de la sesión

   AUTH (admin/profesor):
     POST /api/auth/login                   → { nombre, pin } → cookie JWT
     GET  /api/auth/me                     → datos del admin logueado
     POST /api/auth/logout                 → limpiar cookie

   ADMIN (requiere cookie JWT — requireProfesor en escritura):
     GET    /api/categorias                → listar
     POST   /api/categorias                → crear { nombre, icono }
     PUT    /api/categorias/:id            → editar
     DELETE /api/categorias/:id            → soft delete (cascade a niveles y flashcards)
     GET    /api/niveles                   → listar (?categoria_id)
     POST   /api/niveles                   → crear { nombre, categoria_id }
     PUT    /api/niveles/:id               → editar { nombre }
     DELETE /api/niveles/:id               → soft delete
     GET    /api/flashcards                → listar (?categoria_id&nivel_id&palabra&page&limit)
     GET    /api/flashcards/:id            → detalle
     POST   /api/flashcards                → crear { palabra, descripcion, imagen_url, video_url, categoria_id, nivel_id }
     PUT    /api/flashcards/:id            → editar
     DELETE /api/flashcards/:id            → soft delete
     GET    /api/alumnos                   → listar alumnos (legacy)
     POST   /api/alumnos                   → crear alumno (legacy)
     DELETE /api/alumnos/:id               → soft delete (legacy)

   ANALÍTICAS GLOBALES (requiere auth):
     GET /api/progreso/analiticas          → stats basadas en Sesion (sesiones anónimas):
       { totalAlumnos (sesiones), alumnosActivos (7d), sesionesHoy, palabrasTotales,
         calidadFlashcards, actividadSemanal, distribucionCalif }

   CLOUDINARY (requiere auth + profesor):
     GET  /api/upload/status               → { configurado, cloud_name }
     POST /api/upload/signature            → { signature, timestamp, cloud_name, api_key, folder }

   HEALTH:
     GET /api/health                       → { status, modulo, database }

6. FLUJO DE LA APP (público)

   1. Niño abre la app → ve Inicio con juegos disponibles
   2. Selecciona categoría y juego → juega
   3. Cada acierto/fallo se registra en sesión anónima (sessionId en sessionStorage, deviceId en localStorage)
   4. Al terminar un juego → se registra juego completado
   5. Las stats se muestran en Inicio y FinSesion
   6. Profesor accede vía /admin-login → gestiona contenido y ve analíticas globales

    --- Mejora Admin: Niveles agrupados por categoría ---
    AdminNiveles.tsx: Los niveles ahora se muestran agrupados por categoría
    en cards separadas con header (ej: "🐕 Animales" con sus niveles dentro,
    luego "🍕 Comida" con los suyos). Se usa useMemo para agrupar niveles
    por categoria_id. El formulario de crear mantiene el selector de categoría.

    --- Optimización 1: Code Splitting con lazy loading ---
    routes.tsx: Todas las importaciones de páginas cambiaron de static a
    React.lazy(() => import(...)). Cada página es ahora un chunk separado
    que se descarga solo cuando el usuario navega a ella.
    App.tsx: RouterProvider envuelto en <Suspense fallback={<PageLoader />}>
    PageLoader.tsx: Nuevo componente simple (🤟 animado + "Cargando...").

    Impacto: navegación a /diccionario y otras páginas pasa de ~2s a
    instantáneo (~100-200ms) porque el bundle principal se reduce de
    ~500 KB a ~50 KB. Los chunks de página se descargan bajo demanda.

    --- Optimización 2: JuegoUnir — nuevo flujo "seña primero → palabra después" ---
    ANTES: Videos autoplay simultáneos en todas las señas. Niño veía imagen del
    objeto en la columna izquierda y podía conectar directamente. Videos pesados
    con thumbnail Cloudinary de baja calidad.

    AHORA:
    - La columna derecha (señas) muestra inicialmente solo 🤟 "Toca para ver"
      (sin imagen, sin video). El niño no puede adivinar la respuesta visualmente.
    - Al tocar una seña: se carga y reproduce SOLO ese video (calidad original,
      sin thumbnails). Es el ÚNICO video activo en todo momento.
    - Mientras el video carga, la columna izquierda (palabras) muestra
      "⏳ Preparando..." y los botones están desactivados.
    - Cuando el video está listo (onLoadedData), las palabras se activan y
      el niño toca la palabra que cree correcta.
    - Si acierta: animación green pulse 400ms + Check ✅ + ambos se marcan
      como conectados. El video se reemplaza por imagen estática.
    - Si falla: feedback rojo 600ms + "❌ Intenta otra" en la palabra, el
      video SIGUE sonando y el niño puede tocar otra palabra. Sigue intentando
      hasta acertar.
    - Al completar todos los pares, navega a /juegos/fin con stats.

    Rendimiento: solo 1 video cargado a la vez vs antes 5 simultáneos.

    --- Calidad original restaurada en todos los juegos ---
    JuegoFlashcards.tsx: Se eliminó getThumbnailUrl. Las opciones usan
    option.videoUrl directo (calidad original 200-400 KB por video).
    JuegoQuiz.tsx: Se eliminó getThumbnailUrl. Usa currentPalabra.videoUrl
    directo.
    src/app/lib/cloudinary.ts: Se mantiene el helper getThumbnailUrl por si
    se necesita en el futuro, pero no se importa en ningún juego.

    --- Optimización 3: JuegoFlashcards — 1 video a la vez con preview ---
    ANTES: 4 videos autoPlay muted loop simultáneos en grid 2×2. El
    decodificador se saturaba y los videos se ponían bruscos a los pocos
    segundos. Skeleton mientras cargaban (poco útil si todos cargan a la vez).

    AHORA: Cada opción muestra 🤟 genérico (placeholder estático, sin video).
    Debajo de cada una hay 2 botones:
      [👁️ Ver]  → carga y reproduce SOLO ese video en la tarjeta
      [✓ Elegir] → selecciona esta como respuesta sin necesidad de ver video
    Cuando un video está activo, los botones cambian a:
      [✓ Elegir] [✕ Cerrar]  → elegir esa respuesta o cerrar preview
    Solo 1 video activo en todo momento (si abres otro preview, el anterior
    se cierra). El video se remonta con key={previewId} para reinicio limpio.

    --- Optimización 4: JuegoMemoria — sin video en cartas acertadas + transiciones suaves ---
    ANTES: Cuando un par se acertaba, isMatched=true pero el <video> seguía
    reproduciéndose con autoPlay muted loop. 3 pares acertados = 6 cartas con
    video = 3 videos corriendo sin parar. Cuando fallaban, las cartas se
    volteaban de regreso en 1.5s y el video se cortaba en seco.

    AHORA: 
    - Cuando isMatched: la carta muestra solo ✅ + texto de la palabra (sin video).
      El video solo se renderiza cuando isFlipped && !isMatched.
    - Timeouts aumentados: acierto → 2.5s, fallo → 2.5s (antes 2s y 1.5s).
      El niño ve el par por más tiempo antes de que pase al siguiente estado.
    - Contenido revelado envuelto en <AnimatePresence> con exit animation
      (fade + scale 0.8) para que al fallar el video se desvanezca suavemente
      en vez de desaparecer instantáneo.
    - layout prop en motion.button para transiciones fluidas de layout.

    --- Mejora Diccionario: filtro de niveles dinámico ---
    Diccionario.tsx: 
    - ANTES: el dropdown de niveles mostraba todos los niveles de todas las
      categorías → "Básico, Básico, Básico..." repetido.
    - AHORA: el filtro de nivel se deshabilita cuando "Todas las categorías"
      está seleccionado (placeholder: "Elige una categoría primero").
    - Al seleccionar una categoría específica, el dropdown de nivel se activa

    - Se usa useMemo para filtrar niveles por categoria_id.
    - Al cambiar de categoría, el nivel se resetea a "todos".

    Problema: cada video pesa ~10 MB en disco. 5000 × 10 MB = 50 GB,

    Solución: Se agregaron parámetros de compresión a la subida:
    quality: "auto:good" (compresión perceptual sin pérdida visible)
    bitrate: "500k" (bitrate máximo controlado)
